# 2048 — десктопное приложение с математическим ядром

Проект "под ключ": от алгоритмического ядра до контейнеризации.
Java 17 + JavaFX, файловое хранилище (JSON), Docker.

## Концепция

Классическая игра 2048 с двумя ключевыми алгоритмическими компонентами:

1. **Алгоритм сдвига и слияния тайлов** — детерминированная логика игры (`engine/GameBoard`).
2. **Эвристический ИИ (expectiminimax)** — автоигра с поиском по дереву ходов
   с учётом случайности появления новых тайлов (`ai/AIPlayer`, `ai/Evaluator`).

Дополнительно: рекорды, статистика достигнутых тайлов, режим автоигры —
всё хранится в JSON-файлах, приложение упаковано в Docker-образ.

## Архитектура

```
com.game2048
├── engine/              — математическое ядро игры
│   ├── GameBoard.java   — сдвиг, слияние, спавн тайлов, проверка конца игры
│   ├── Move.java        — enum направлений
│   └── GameState.java   — снимок партии для сохранения/загрузки
├── ai/                  — алгоритмический ИИ-модуль
│   ├── AIPlayer.java    — expectiminimax (поиск по дереву ходов)
│   └── Evaluator.java   — эвристическая функция оценки позиции
├── data/                — файловое хранилище (без БД/СУБД)
│   ├── GameRecord.java  — запись завершённой партии
│   └── DataManager.java — чтение/запись JSON (Gson)
├── ui/                  — JavaFX-интерфейс
│   ├── GameApp.java     — точка входа, оркестрация модулей
│   ├── BoardView.java   — отрисовка доски
│   └── StatsPanel.java  — счёт, рекорд, статистика
└── Main.java            — launcher (нужен для корректной сборки fat-jar)
```

Принцип: **`engine` и `ai` не знают о UI и файлах** — их можно тестировать
и переиспользовать независимо (см. `src/test`). `ui` и `data` — тонкие
адаптеры вокруг ядра.

## Алгоритмы подробно

### 1. Сдвиг и слияние (`GameBoard`)

Любое из четырёх направлений сводится к одной операции — "сдвиг влево":

- `RIGHT` = зеркальный разворот строк → сдвиг влево → разворот обратно
- `UP` = транспонирование матрицы → сдвиг влево → транспонирование обратно
- `DOWN` = транспонирование + разворот → сдвиг влево → разворот + транспонирование

Сама операция `mergeRow`:
1. Убрать нули (компрессия).
2. Пройти слева направо: если два соседних тайла равны — слить (значение ×2,
   тайл участвует в слиянии не более одного раза за ход — классическое правило 2048).
3. Дополнить нулями справа.

Сложность: O(n) на строку, O(n²) на всю доску (n=4).

### 2. ИИ — expectiminimax (`AIPlayer` + `Evaluator`)

2048 — игра со случайностью (после хода игрока появляется случайный тайл),
поэтому классический минимакс не подходит напрямую. Используется
**expectiminimax** с двумя типами узлов дерева:

- **MAX-узел** (ход игрока): перебираются 4 направления, выбирается то,
  что даёт максимальную ожидаемую оценку следующего состояния.
- **CHANCE-узел** (появление тайла): оценка усредняется по всем пустым
  клеткам и обоим значениям тайла, взвешенная по вероятности
  (2 — вероятность 0.9, 4 — вероятность 0.1).

Глубина поиска **адаптивная**: чем меньше свободных клеток на доске, тем
меньше ветвлений в CHANCE-узлах — можно позволить себе искать глубже
без экспоненциального взрыва числа состояний.

Эвристика оценки позиции (`Evaluator`) комбинирует:
- **Позиционный вес** (snake pattern) — крупные тайлы поощряются в углу;
- **Число свободных клеток** — больше пространства для манёвра;
- **Гладкость** — штраф за резкие перепады значений соседних тайлов.

### 3. Файловое хранилище (`DataManager`)

Каталог `~/.game2048/`:
- `records.json` — история всех завершённых партий (список `GameRecord`);
- `current_game.json` — снимок незавершённой партии (для восстановления
  при перезапуске).

Статистика (рекорд, гистограмма тайлов, число партий) вычисляется "на лету"
поверх `records.json` — отдельный агрегатный файл не нужен.

## Сборка и запуск (локально)

Требуется JDK 17+ и Maven.

```bash
# Запуск через плагин (для разработки)
mvn clean javafx:run

# Сборка fat-jar
mvn clean package
java -jar target/game2048-1.0.0.jar

# Запуск тестов
mvn test
```

## Запуск в Docker

Так как приложение — GUI (JavaFX), контейнеру нужен проброс X11-дисплея хоста.

**Linux:**
```bash
xhost +local:docker
docker compose up --build
```

**Windows/macOS:** потребуется X-сервер (VcXsrv для Windows, XQuartz для macOS)
и соответствующая настройка `DISPLAY` в `docker-compose.yml`. Альтернатива —
собрать и протестировать `engine`/`ai`/`data` модули в контейнере без GUI
(`mvn test` внутри контейнера), а само приложение запускать локально.

Данные (рекорды, статистика) сохраняются в именованном volume
`game2048-data` — переживают пересборку и удаление контейнера.

## Управление

- Стрелки / WASD — ход
- «Новая игра» — сброс партии
- «Автоигра (ИИ)» — переключатель режима: дальше ходит ИИ (expectiminimax)
